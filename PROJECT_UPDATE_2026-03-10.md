# 项目更新记录（2026-03-10）

## 一、今日新增与调整

### 1. 分析主链路已切换为音频转录优先

当前系统不再以“Gemini 直接理解整段视频”作为主流程，而是改成：

`上传视频 -> 分离音频 -> OpenAI 转录 -> Gemini 基于转录文本分析 -> 输出会议结果`

本次改动的意义：

- 更适合会议纪要、老板要求、员工汇报、待办事项、决策点这类以语义为核心的场景
- 相比直接分析视频，成本更容易控制
- 后续更容易做发言人、任务状态、历史联动

相关实现：

- [audio_transcriber.py](/Users/saita/saita/videos-agnets/processors/audio_transcriber.py)
- [structured_transcript.py](/Users/saita/saita/videos-agnets/processors/structured_transcript.py)
- [meeting_pipeline.py](/Users/saita/saita/videos-agnets/meeting_pipeline.py)

### 2. 新增结构化转录能力

系统现在会先生成结构化会议转录，而不是只保留一段普通 transcript。

结构化信息包括：

- 完整转录文本
- `utterances` 发言片段
- 发言人 ID
- 发言人姓名/角色
- 时间信息

转录结果会保存为：

- `transcription.json`
- `structured_transcript.json`
- `participant_roles.json`

### 3. Web 端已统一为“完整分析”

之前存在“快速概览”和“完整分析”两条入口。现在已经移除快速概览，只保留完整分析。

这样做的原因：

- 避免双流程维护
- 降低异常分支
- 所有结果统一沉淀到历史和数据库中

### 4. 页面改成联动工作台

现在分析、查看、回看、下载不再跳到割裂的独立结果页，而是在同一页面联动完成。

当前 Web 页面的工作方式：

- 左侧显示历史记录
- 右侧显示当前会议结果
- 页内切换员工汇报、议题、待办、发言人、决策、完整报告

相关实现：

- [app.py](/Users/saita/saita/videos-agnets/web/app.py)
- [index.html](/Users/saita/saita/videos-agnets/web/templates/index.html)

### 5. 历史记录已接入数据库并支持删除

每次分析结果不会丢失，系统会保留：

- 数据库记录
- 输出目录文件
- 前端历史入口

同时已经支持在前端删除历史记录，删除时会同步清理：

- 数据库中的该次会议数据
- 本地输出目录

### 6. 新增长期记忆层（任务总表 + 要求追踪）

系统已经不再只做“单次会议总结”，而是开始把会议信息沉淀到长期记忆层。

目前已落地：

- 全局任务总表 `task_memory`
- 任务更新历史 `task_updates`
- 老板要求库 `requirement_memory`
- 要求更新历史 `requirement_updates`

当前长期记忆会记录：

- 任务标题
- 负责人
- 当前状态
- 优先级
- 截止日期
- 来源会议
- 首次提出日期
- 最近提及会议
- 完成会议
- 被提及次数
- 最近一次摘要和上下文

相关实现：

- [database.py](/Users/saita/saita/videos-agnets/knowledge/database.py)

### 7. 新增跨会议任务状态联动

这是今天最重要的增强之一。

现在每次新会议分析结束后，系统会用本次会议转录去对比历史任务总表，自动识别任务在本次会议中的状态变化，并回写到全局任务中心。

当前支持识别的状态：

- `progress` 推进中
- `completed` 已完成
- `blocked` 阻塞
- `reopened` 重新打开

回写后，任务中心会显示：

- 任务最早在哪次会议提出
- 哪天提出
- 最近在哪次会议再次被提到
- 如果已完成，是在哪次会议确认完成

- [meeting_pipeline.py](/Users/saita/saita/videos-agnets/meeting_pipeline.py)
- [database.py](/Users/saita/saita/videos-agnets/knowledge/database.py)

### 8. 新增独立的“全局任务中心”视图

前端已经不再只把任务记忆塞在单次会议详情里，而是增加了独立的全局任务中心，供老板或管理者直接查看。

当前全局任务中心支持查看：

- 全局任务数量
- 待跟进 / 推进中 / 已完成 / 阻塞 数量
- 每个任务的负责人、优先级、截止日期
- 来源会议
- 提出日期
- 最近提及会议
- 完成会议和完成日期
- 最近几次更新历史

同时老板要求库也保留在任务中心内，作为长期追踪的辅助区域。

### 9. 旧历史数据已回填长期记忆层

为了避免只对新会议生效，系统已经对现有历史会议做了一次回填，把旧 action items 同步进了新的全局任务表。

## 二、当前系统总体实现的功能

### 1. 视频上传与会议分析

用户可以在 Web 端上传会议视频，系统会自动完成：

- 视频接收与存储
- 音频提取
- 语音转录
- 文本结构化
- 会议分析

### 2. 结构化会议理解

系统会从会议中提取：

- 会议标题
- 摘要
- 关键点
- 分类
- 参与者
- 发言片段
- 管理视角摘要

### 3. 管理视角总结

系统目前已经支持管理视角输出，包括：

- 老板要求
- 员工汇报摘要
- 风险与阻塞
- 下一步动作

### 4. 议题时间轴

系统会把会议拆成多个议题段，并生成议题时间轴，包含：

- 议题标题
- 开始时间 / 结束时间
- 议题摘要
- 议题关键点

### 5. 待办事项提取

系统会从会议内容中提取待办事项，包含：

- 任务内容
- 负责人
- 截止日期
- 优先级
- 任务来源上下文

### 6. 发言人汇总

系统可以基于结构化转录汇总发言人信息，包括：

- 发言人 ID
- 发言人姓名
- 角色
- 发言量
- 参与度

### 7. 决策点识别

系统可以从会议中提取关键决策，包括：

- 决策标题
- 决策描述
- 决策原因
- 影响等级
- 关联议题

### 8. 多格式导出

目前支持生成：

- Markdown 会议纪要
- HTML 时间轴报告
- PDF 报告
- 结构化转录 JSON

### 9. 历史记录管理

系统已支持会议历史记录保留与回看：

- 每次分析自动入历史
- 可从左侧历史列表打开旧会议
- 可下载对应产物
- 可删除历史记录

### 10. 长期记忆与跨会议联动

系统已经具备基础的长期记忆能力：

- 会议中的待办事项会沉淀到全局任务中心
- 后续会议会尝试识别这些任务是否推进、完成或阻塞
- 任务可关联到来源会议和完成会议
- 老板要求会进入长期要求库

### 11. 当前 Web 页面结构

目前 Web 端主要分成两个视图：

#### 会议工作台

用于：

- 上传视频
- 启动分析
- 查看单次会议结果
- 下载结果文件
- 回看历史会议

#### 全局任务中心

用于：

- 查看全局任务状态
- 查看任务来源会议和时间
- 查看截止日期
- 查看任务完成归档
- 查看老板要求追踪

## 三、当前主流程

当前系统的完整主流程如下：

1. 用户上传会议视频
2. 系统提取视频音频
3. 调用 OpenAI 转录接口生成文字和发言片段
4. 调用 Gemini 对转录文本进行结构化分析
5. 提取老板要求、员工汇报、风险、下一步
6. 生成会议分段、待办事项、发言人、决策点
7. 生成 Markdown / HTML / PDF / JSON 结果
8. 保存会议历史到数据库
9. 同步到全局任务中心和老板要求库
10. 用本次会议内容更新历史任务状态
11. 在前端工作台和全局任务中心中展示结果

## 四、当前已知边界

### 1. 任务状态联动已经可用，但仍属于第一版

现在的任务跨会议更新是基于模型理解来判断的，已经可用，但还不是完全规则化的强约束系统。

这意味着：

- 对“明显完成”“明显阻塞”的任务判断会比较有效
- 对说法模糊、表达绕的会议内容，仍可能需要人工抽查

### 2. 老板要求库和任务总表还不是完全同一概念

目前系统里：

- `任务总表` 更偏执行项、待办、进展
- `老板要求库` 更偏长期要求、督办、反复强调事项

后续如果业务上确认两者应该彻底合并，可以继续收敛数据模型。

### 3. Gemini SDK 仍需后续迁移

当前代码还在使用已弃用的 `google.generativeai` SDK，虽然目前可运行，但后续建议迁移到 `google.genai`。

## 五、建议的下一步

后续最值得继续做的方向：

1. 提升任务匹配精度  
让“旧任务被重新提到”的识别更稳，减少语义漂移。

2. 增加任务筛选与搜索  
支持按负责人、状态、截止日期、项目筛选全局任务。

3. 增加项目维度  
把任务按项目归集，而不只是按会议归集。

4. 增加任务手动修正能力  
允许管理者在前端手动把任务改成已完成、阻塞、延期。

5. 迁移 Gemini SDK  
把当前模型调用从 `google.generativeai` 迁到 `google.genai`。
